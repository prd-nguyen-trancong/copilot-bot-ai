# copilot-bot-ai — Developer Documentation

A VS Code extension that sends prompts directly to GitHub Copilot models (no Chat UI required)
and streams the response back as text. It also ships a terminal CLI (`copilot-bot`) that works
in any macOS or Ubuntu shell after a one-time VS Code sign-in.

## VS Code Extension API

### Language Model API

The extension uses VS Code's built-in `vscode.lm` namespace, available since VS Code 1.91.

#### Selecting a model

```typescript
const [model] = await vscode.lm.selectChatModels({
  vendor: 'copilot',
  family: 'gpt-4o',   // e.g. 'claude-sonnet-4.6', 'o3-mini'
});
```

`selectChatModels` returns all models that match the selector. The first match is used.
If the desired family is unavailable, `CopilotService` falls back to the first available model.

#### Sending a request and streaming

```typescript
const messages = [
  vscode.LanguageModelChatMessage.User('Your prompt here'),
];

const chatResponse = await model.sendRequest(messages, {}, cancellationToken);

// chatResponse.text is AsyncIterable<string>
for await (const fragment of chatResponse.text) {
  outputChannel.append(fragment);   // stream each chunk
}
```

`sendRequest` returns immediately; text arrives asynchronously in chunks.

#### Cancellation

Pass a `vscode.CancellationToken` as the third argument to `sendRequest`.
Create tokens with `new vscode.CancellationTokenSource()`.

```typescript
const cts = new vscode.CancellationTokenSource();
const response = await model.sendRequest(messages, {}, cts.token);
// Later:
cts.cancel();
```

### Public API for Other Extensions

`copilot-bot-ai` exports a typed API so other VS Code extensions can call it directly.

**In the consuming extension's package.json:**

```json
{
  "extensionDependencies": ["n.tran.copilot-bot-ai"]
}
```

**TypeScript usage:**

```typescript
import type { CopilotBotAiApi } from 'copilot-bot-ai/src/types';

const ext = vscode.extensions.getExtension<CopilotBotAiApi>('n.tran.copilot-bot-ai');
const api = await ext!.activate();

// Stream a response
const result = await api.sendPrompt(
  {
    prompt: 'Explain event-driven architecture in 3 sentences.',
    modelFamily: 'claude-sonnet-4.6',
    systemPrompt: 'You are a senior software engineer who explains things simply.',
  },
  (chunk) => process.stdout.write(chunk)
);
console.log('Full response:', result.fullText);
console.log('Model used:',   result.model);
```

**`CopilotRequestOptions` fields:**

| Field | Type | Required | Description |
|---|---|---|---|
| `prompt` | `string` | yes | The user prompt |
| `modelFamily` | `string` | no | Override model family. Falls back to config setting. |
| `systemPrompt` | `string` | no | System instruction prepended to the conversation |
| `cancellationToken` | `vscode.CancellationToken` | no | Token to cancel the request |

**`CopilotResponse` fields:**

| Field | Type | Description |
|---|---|---|
| `fullText` | `string` | Complete assembled response text |
| `model` | `string` | The model ID that was actually used |

## CLI Tool

`bin/copilot-bot.mjs` calls the GitHub Copilot REST API directly using a token saved by the
VS Code extension.  It works in any terminal that has Node.js >= 18 installed.

### One-Time Auth Setup

1. Open VS Code with the `copilot-bot-ai` extension installed.
2. Open the Command Palette: **Cmd+Shift+P** (macOS) or **Ctrl+Shift+P** (Ubuntu).
3. Run: **Copilot Bot AI: Save Token for CLI**
4. VS Code will prompt you to sign in to GitHub if not already signed in.
5. The token is written to `~/.copilot-bot-ai/config.json` (permissions 0600 — readable only by you).

Repeat this step only if your token expires or you change GitHub accounts.

### Usage

```bash
# Basic prompt
copilot-bot "What is a monad?"

# Choose a specific model
copilot-bot "Explain async/await in JavaScript" --model claude-sonnet-4.6

# Custom system prompt
copilot-bot "Review this code" \
  --system "You are a senior TypeScript engineer. Be concise and critical."

# Pipe from stdin
echo "Write a haiku about TypeScript" | copilot-bot

# Process a file
copilot-bot "Summarise this" < ./long_document.txt | tee summary.txt

# Use in a script
RESULT=$(copilot-bot "Generate a UUID" --model gpt-4o-mini)
echo "Generated: $RESULT"
```

**Flags:**

| Flag | Default | Description |
|---|---|---|
| `--model <name>` | `gpt-4o` | Model family to use |
| `--system <text>` | (none) | Custom system prompt |
| `--help`, `-h` | — | Print usage and exit |

## Packaging as .vsix

A `.vsix` file is a self-contained archive that anyone can install without the Marketplace.

### Install the packaging tool

```bash
# Option A — global install
npm install -g @vscode/vsce

# Option B — use npx (no global install needed)
npx @vscode/vsce package
```

### Create the .vsix

```bash
npm run compile          # ensure latest build
vsce package             # creates copilot-bot-ai-<version>.vsix
```

Or use the combined npm script:

```bash
npm run package          # = npm run compile && vsce package
```

The `.vsix` appears in the project root, e.g. `copilot-bot-ai-1.0.0.vsix`.

## Installing Locally

### GUI

1. Open VS Code.
2. Open the Extensions panel (Cmd+Shift+X / Ctrl+Shift+X).
3. Click the `...` menu → **Install from VSIX...**
4. Select the `.vsix` file.

### Command line

```bash
code --install-extension copilot-bot-ai-1.0.0.vsix
```

### Package and install in one step

```bash
npm run package:install    # = vsce package && code --install-extension copilot-bot-ai-*.vsix
```

### Make the CLI available system-wide (after extension install)

```bash
# From the project directory:
npm link              # symlinks `copilot-bot` into your PATH
# — OR —
npm install -g .      # copies to global node_modules
```

## Sharing & Distribution

### Share via .vsix File

1. Run `npm run package` to produce the `.vsix`.
2. Send the file to users (email, Slack, GitHub Releases, etc.).
3. Recipients install it:

   ```bash
   code --install-extension copilot-bot-ai-<version>.vsix
   ```

4. They run the one-time auth step:
   - Command Palette → **Copilot Bot AI: Save Token for CLI**
5. `copilot-bot` is then available in any terminal (after `npm link` or `npm install -g .`).

### Publish to VS Code Marketplace

Publishing lets users install the extension with one click from the Extensions panel.

#### Step 1 — Create a publisher account

- Go to [marketplace.visualstudio.com/manage](https://marketplace.visualstudio.com/manage)
- Sign in with a Microsoft account.
- Click **Create publisher** and choose a unique publisher ID (e.g. `yourname`).

#### Step 2 — Create a Personal Access Token (PAT)

- Visit [dev.azure.com](https://dev.azure.com) → User settings → **Personal Access Tokens**
- Click **New Token**
- Set **Organization** to "All accessible organizations"
- Set **Scope** to **Marketplace → Manage**
- Copy the token immediately (it is shown only once)

#### Step 3 — Login with vsce

```bash
vsce login <your-publisher-id>
# Paste your PAT when prompted
```

#### Step 4 — Set publisher in package.json

```json
{
  "publisher": "your-publisher-id"
}
```

#### Step 5 — Publish

```bash
vsce publish           # auto-increments patch version and publishes
vsce publish minor     # increments minor version
vsce publish major     # increments major version
vsce publish 1.2.0     # publish with explicit version
```

After publishing, the extension is available at:

```
https://marketplace.visualstudio.com/items?itemName=<publisher>.copilot-bot-ai
```

Users can then install it with:

```bash
code --install-extension <publisher>.copilot-bot-ai
```

## Development Workflow

### Debug in VS Code Extension Host

1. Open this project in VS Code.
2. Press **F5** (Run → Start Debugging).
3. A second VS Code window opens with the extension loaded.
4. Command Palette → **Copilot Bot AI: Send Prompt** to test.
5. Check **Output** panel → channel **Copilot Bot AI** for streamed responses.

```bash
npm run watch
```

```bash

```

## Troubleshooting

| Problem | Fix |
|---|---|
| `No auth token found` in CLI | Run **Copilot Bot AI: Save Token for CLI** in VS Code |
| `API error 401` in CLI | Token expired — re-run the save command |
| `No language model available` | Ensure GitHub Copilot is active and you are signed in |
| `copilot-bot: command not found` | Run `npm link` or `npm install -g .` in the project directory |
| Extension not loading | Check VS Code >= 1.99.0 and run `npm run compile` |
| TypeScript errors after `npm install` | Delete `out/` and run `npm run compile` again |
